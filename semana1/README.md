Antes de estudiar SOLID pensaba que una clase debía contener todo lo necesario para que un sistema funcionara.
Durante el ejercicio entendí que una clase debe tener una responsabilidad clara.
Por ejemplo, en un sistema de monitoreo no es buena idea que una sola clase lea sensores, procese datos, guarde archivos y envíe información por WiFi.
Separar responsabilidades permite modificar una parte sin afectar todo el sistema.
Además, SOLID nos ayuda a escribir código más limpio y fácil de mantener.
